LED-03: Status-LED erst nach 30s Disconnect aktivieren statt sofort.

**1. Neue globale Variablen (nach Zeile 190, bei den WiFi-Reconnect-Variablen, nach `wifiReconnectActive`):**
```cpp
unsigned long disconnectStartMs = 0;                // LED-03: Grace-Timer fuer Status-LED
const unsigned long STATUS_LED_GRACE_MS = 30000;     // LED-03: 30s bevor Status-LED aktiviert wird
```

**2. WiFi-Disconnect-Handler (Zeile 3742-3748) — setStatusLED(1) entfernen, Timer starten:**
Ersetze `setStatusLED(1);` (Zeile 3748) mit:
```cpp
disconnectStartMs = millis();  // LED-03: Grace-Timer starten, KEIN setStatusLED(1) hier
```
Falls `millis()` 0 zurueckgibt (direkt nach Boot): kein Problem, da der ESP zu dem Zeitpunkt noch nicht WiFi-connected war.

**3. Grace-Period-Pruefung im WiFi-Check (nach dem `if (wifiWasConnected)` Block, aber INNERHALB des `if (WiFi.status() != WL_CONNECTED)` Blocks, vor dem Reconnect-Attempt-Block):**
Fuege ein:
```cpp
// LED-03: Status-LED erst nach Grace Period aktivieren
if (disconnectStartMs > 0 && (currentMillis - disconnectStartMs > STATUS_LED_GRACE_MS)) {
    setStatusLED(1);
    disconnectStartMs = 0;  // Einmalig setzen
}
```
Platzierung: Nach dem `if (wifiWasConnected) { ... }` Block (Zeile ~3749), vor dem `if (currentMillis - lastWifiCheck >= wifiReconnectDelay)` Block (Zeile ~3752).

**4. Bei erfolgreichem Reconnect (Zeile 3779-3794) — Timer zuruecksetzen:**
Nach `wifiReconnectActive = false;` (aus Task 1) einfuegen:
```cpp
disconnectStartMs = 0;  // LED-03: Grace-Timer zuruecksetzen
```
`setStatusLED(0);` bleibt wie gehabt (Zeile 3794).

**5. Im `else if (!wifiWasConnected)` Block (Zeile 3805+) — Timer zuruecksetzen:**
Nach `wifiReconnectActive = false;` einfuegen:
```cpp
disconnectStartMs = 0;  // LED-03: Grace-Timer zuruecksetzen
```
