MEM-02: Zwei Health-Check-Timer zu einem konsolidierten Ablauf zusammenfuehren.

**Problem:** Es gibt zwei Bloecke die `sysHealth.isRestartRecommended()` aufrufen:
1. 1h-Block (Zeile 4366): Plant Neustart via Nacht-Logik (korrekt)
2. 5min-Block (Zeile 4490): Setzt `rebootNeeded = true` sofort (BUG — kein Nacht-Check)

**Loesung:** Den 5min-Block (Zeilen 4488-4500) auf reine Update-Aufrufe reduzieren und den Restart-Trigger entfernen.

**1. Den 5min-Block (Zeilen 4488-4500) ersetzen mit:**
```cpp
// Regelmaessiger Systemgesundheitscheck (5 Minuten)
static unsigned long lastHealthCheck = 0;
if (millis() - lastHealthCheck > 300000) {
    sysHealth.update();
    lastHealthCheck = millis();
}
```

Entfernt wird:
- `if (sysHealth.isRestartRecommended()) { ... }` (Zeilen 4494-4499) — der Restart-Trigger ohne Nacht-Logik
- `rebootNeeded = true;` und `rebootTime = millis() + 60000;` (Zeilen 4497-4498)

**2. Den `memMonitor.update();` Aufruf (Zeile 4486) in den 5min-Block verschieben:**
Der aktuelle `memMonitor.update();` steht AUSSERHALB jedes Timers (Zeile 4486) und wird in JEDEM loop()-Durchlauf aufgerufen — das ist unnoetig haeufig. Verschiebe ihn in den 5min-Block:

```cpp
// Regelmaessiger Systemgesundheitscheck (5 Minuten)
static unsigned long lastHealthCheck = 0;
if (millis() - lastHealthCheck > 300000) {
    sysHealth.update();
    memMonitor.update();
    lastHealthCheck = millis();
}
```

Entferne die freistehende Zeile `memMonitor.update();` (Zeile 4486).

**3. Im 1h-Block (Zeile 4366-4472): `sysHealth.update()` VOR `isRestartRecommended()` aufrufen:**
Am Anfang des 1h-Blocks (nach Zeile 4367 `debug(...)`) einfuegen:
```cpp
sysHealth.update();
```
Damit ist sichergestellt, dass die Health-Daten aktuell sind bevor die Restart-Entscheidung getroffen wird.

**Ergebnis:**
- 5min: `sysHealth.update()` + `memMonitor.update()` (kein Restart-Trigger)
- 1h: Stats schreiben + `sysHealth.update()` + `isRestartRecommended()` mit Nacht-Logik (einziger Restart-Pfad)
