3. In `sentiment-api/app.py` den `start_background_worker()` Aufruf verschieben:

KRITISCH: Der Aufruf liegt aktuell innerhalb des `if __name__ == '__main__':` Blocks. Gunicorn fuehrt `__main__` NICHT aus — der Worker wuerde nie starten.

- Den `start_background_worker(...)` Aufruf (mit allen Parametern wie app, analyze_function, interval_seconds) aus dem `if __name__ == '__main__':` Block HERAUS nehmen
- Ihn auf Modul-Level platzieren, NACH `register_moodlight_endpoints(app)` und VOR dem `if __name__ == '__main__':` Block
- Einen Kommentar hinzufuegen: `# Background Worker starten - muss ausserhalb __main__ sein damit Gunicorn ihn startet`
- Den Aufruf aus dem `if __name__ == '__main__':` Block ENTFERNEN (nicht doppelt!)
- Der `if __name__ == '__main__':` Block behaelt nur noch `app.run(...)` fuer lokale Entwicklung

Ergebnis-Struktur:
```python
register_moodlight_endpoints(app)

# Background Worker starten - muss ausserhalb __main__ sein damit Gunicorn ihn startet
start_background_worker(
    app=app,
    analyze_function=analyze_headlines_openai_batch,
    interval_seconds=1800
)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=6237, debug=False)
```

WARNUNG: Die genauen Parameter von start_background_worker() aus dem bestehenden Code uebernehmen — nicht raten.
