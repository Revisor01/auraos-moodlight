Änderungen im Inline-Script (der komplette Script-Block am Ende der Datei):
1. Die Funktion `scoreColor(score)` umbenennen in `scoreClass(score)` und statt Hex-Farben CSS-Klassen-Strings zurückgeben:
   ```js
   function scoreClass(score) {
       if (score <= -0.4) return 'score-sehr-negativ';
       if (score < 0)     return 'score-negativ';
       if (score === 0)   return 'score-neutral';
       if (score < 0.4)   return 'score-positiv';
       return 'score-sehr-positiv';
   }
   ```
2. In `renderHeadlines()` den HTML-String für jede Schlagzeile anpassen:
   - Statt `style="color:' + color + '"` → `class="' + scoreClass(h.sentiment_score) + '"`
   - Statt `style="display:flex;align-items:baseline;gap:8px;padding:6px 0;border-bottom:1px solid rgba(128,128,128,0.15);"` → class="headline-item"
   - Die resultierenden Zeilen:
     ```js
     html += '<div class="headline-item">';
     html += '<span class="' + cls + ' headline-score">' + score + '</span>';
     html += '<span class="headline-text">' + h.headline + '</span>';
     if (feed) { html += '<span class="headline-feed">' + feed + '</span>'; }
     html += '</div>';
     ```
   Hinweis: .headline-item, .headline-score, .headline-text, .headline-feed sind neue Hilfsklassen. Da style.css noch keine hat, einen kleinen `<style>`-Block im `<head>` hinzufügen:
   ```html
   <style>
   .headline-item { display:flex; align-items:baseline; gap:8px; padding:6px 0; border-bottom:1px solid var(--border); }
   .headline-score { font-weight:700; min-width:42px; font-size:0.85em; }
   .headline-text  { flex:1; font-size:0.9em; }
   .headline-feed  { font-size:0.75em; color:var(--text-muted); white-space:nowrap; margin-left:6px; }
   </style>
   ```
   Dieser Style-Block ersetzt die vollständig inline kodierten Styles und nutzt CSS-Variablen für Dark-Mode-Kompatibilität.

3. Variable `color` und Funktion `scoreColor` vollständig entfernen, durch `cls = scoreClass(...)` ersetzen.
4. `var color = ... scoreColor(h.sentiment_score) : '#888';` → `var cls = typeof h.sentiment_score === 'number' ? scoreClass(h.sentiment_score) : 'score-neutral';`

Der restliche HTML-Code (Charts, Stats-Grid, Info-Karten, Filter) bleibt strukturell identisch zum Original — keine strukturellen Änderungen an Klassen die mood.js referenziert.
