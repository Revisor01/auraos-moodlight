# Milestone v1.0: AuraOS Moodlight Stabilisierung

**Status:** ✅ SHIPPED 2026-03-25
**Phases:** 1-2
**Total Plans:** 6

## Overview

Stabilitäts-Milestone für das bestehende AuraOS Moodlight System. Keine neuen Features — ausschließlich In-Place-Korrekturen an Firmware (ESP32) und Backend (Flask/Docker). Firmware-Fixes beseitigen LED-Flicker, Buffer-Overflows und Memory Leaks. Backend-Fixes migrieren auf Gunicorn, konsolidieren duplizierte Konfiguration und bereinigen das Repository.

## Phases

### Phase 1: Firmware-Stabilität

**Goal**: Der ESP32 läuft dauerhaft ohne Watchdog-Resets, Heap-Leaks oder unerklärliches LED-Blinken
**Depends on**: Nothing (first phase)
**Requirements**: LED-01, LED-02, LED-03, MEM-01, MEM-02, SEC-01
**Plans**: 3 plans

Plans:
- [x] 01-01: Buffer-Overflow Fix (LED-01) und JSON Buffer Pool RAII Guard (MEM-01)
- [x] 01-02: WiFi-Reconnect LED-Guard (LED-02) und Status-LED Debounce (LED-03)
- [x] 01-03: Health-Check Konsolidierung (MEM-02) und Credential-Maskierung (SEC-01)

**Success Criteria (all verified):**
1. LED-Strip flackert nicht bei WiFi/MQTT-Reconnects
2. Status-LED bleibt bei Unterbrechungen <30s ruhig
3. ESP.getFreeHeap() bleibt nach 24h stabil
4. numLeds > 12 verursacht keinen Crash
5. API-Responses maskieren Passwörter

### Phase 2: Backend-Hardening

**Goal**: Backend läuft produktionstauglich hinter Gunicorn, ohne duplizierte Konfiguration oder Secret-Leaks im Repository
**Depends on**: Phase 1
**Requirements**: BE-01, BE-02, DATA-01, DATA-02, DATA-03, REPO-01, REPO-02
**Plans**: 3 plans

Plans:
- [x] 02-01: Shared Config erstellen (DATA-01, DATA-02) und tote Endpoints entfernen (DATA-03)
- [x] 02-02: Gunicorn-Migration (BE-01) und Per-Connection Timeouts (BE-02)
- [x] 02-03: Repository-Hygiene: .gitignore und getrackte Artefakte (REPO-01, REPO-02)

**Success Criteria (all verified):**
1. Genau ein "Sentiment update" alle 30min in Docker-Logs
2. Score 0.35 = überall "positiv" — konsistente Kategorien
3. RSS-Feed-Liste nur in shared_config.py
4. /api/dashboard und /api/logs → 404
5. .env und *.tmp.html nicht im Git tracked

## Milestone Summary

**Key Decisions:**
- Gunicorn mit `-w 1` (nicht `-w 2`) — Background Worker darf nicht dupliziert werden
- Sentiment-Thresholds 0.30/0.10/-0.20/-0.50 als canonical (nicht die abweichenden app.py Werte)
- MAX_LEDS = 64 als statisches Array (kein dynamisches Heap-Alloc auf ESP32)
- feedparser Timeout via requests.get() statt globalem socket.setdefaulttimeout()

**Issues Resolved:**
- LED-Flicker bei WiFi/MQTT-Reconnects (wifiReconnectActive Flag)
- Buffer-Overflow bei numLeds > 12 (ledColors[MAX_LEDS])
- JSON Buffer Pool Memory Leak bei Mutex-Timeout (release() Fix + RAII Guard)
- Doppelte Health-Check-Timer mit widersprüchlicher Restart-Logik
- Inkonsistente Sentiment-Kategorien zwischen app.py und background_worker.py

**Issues Deferred:**
- esp_task_wdt_init() API-Inkompatibilität mit Arduino ESP32 Core 3.x (pre-existing)
- Firmware-Modularisierung (eigener Milestone)
- HTTPS auf ESP32 (eigener Milestone)

**Technical Debt Incurred:**
- JsonBufferGuard RAII-Klasse existiert aber wird nicht aktiv genutzt (11 manuelle acquire/release-Paare)
- get_category_from_score Import-Alias in moodlight_extensions.py ohne direkten Aufruf
