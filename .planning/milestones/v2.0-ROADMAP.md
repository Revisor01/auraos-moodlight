# Milestone v2.0: Combined Update + Build Automation

**Status:** ✅ SHIPPED 2026-03-26
**Phases:** 3-5
**Total Plans:** 4

## Overview

Build-Workflow-Optimierung: Ein einziges Build-Script erzeugt ein Combined-TGZ (UI + Firmware), bumpt die Version und committet. Der ESP32 verarbeitet das TGZ über zwei neue HTTP-Endpoints — UI-Extraktion nach LittleFS und Firmware-Flash via Streaming. Die Diagnostics-UI hat einen "Full Update"-Button der beides sequentiell ausführt.

## Milestone Summary

**Key Decisions:**
- Two-Route-Architektur: Selbe Datei zweimal hochgeladen (UI-Extraktion + Firmware-Flash) wegen 128KB LittleFS-Limit
- ChunkStream Ring-Buffer mit FreeRTOS-Semaphore für ESP32-targz Streaming-Integration
- firmware.ino.bin Naming-Convention (ESP32-targz erwartet .ino.bin Endung)
- XHR statt fetch für Upload-Progress-Events
- Connection-Abbruch bei Firmware-Flash = Erfolg (ESP restartet)

**Issues Resolved:**
- esp_task_wdt_init API-Inkompatibilität mit Arduino ESP32 Core 3.x (pre-existing aus v1.0)
- build-release.sh || true das Compile-Fehler verschluckte
- Getrennte UI/Firmware-Uploads → ein Combined-Upload
- Manuelle Versionierung → automatischer Version-Bump
- JSON-Parse-Bug in Versionsanzeige (cf29b28)
